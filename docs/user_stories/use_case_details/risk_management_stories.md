## UC-RISK-001: Credit Risk Scoring

### Stakeholder Stories

#### As a Risk Manager
**I want to** assess credit risk for loan applicants
**So that** I can make informed lending decisions and minimize defaults

#### As a Credit Analyst
**I want to** see detailed risk breakdown
**So that** I can understand why a customer is high/low risk

#### As a Compliance Officer
**I want to** ensure risk scoring is fair and explainable
**So that** we meet regulatory requirements

#### As an Admin
**I want to** configure risk thresholds
**So that** the scoring aligns with our risk appetite

## UC-RISK-003: Operational Risk

### Stakeholder Stories

#### As an Operations Manager
**I want to** identify operational risk events
**So that** I can implement preventive controls

#### As a Risk Manager
**I want to** quantify operational risk exposure
**So that** I can allocate appropriate capital

---

### Input/Process/Output

**Input:**
| Field | Type | Description |
|-------|------|-------------|
| Business Unit | Dropdown | Organizational unit |
| Risk Category | Dropdown | People/Process/Systems/External |
| Event Type | Dropdown | Loss type classification |
| Loss Data | File | Historical loss data |
| KRI Data | File | Key Risk Indicator data |

**Process:**
1. Load operational risk data
2. Calculate loss distribution
3. Apply Monte Carlo simulation
4. Calculate operational VaR
5. Generate risk heat map

**Output:**
| Field | Type | Description |
|-------|------|-------------|
| OpRisk VaR | Currency | Operational risk capital |
| Expected Loss | Currency | Annual expected loss |
| Risk Heat Map | Matrix | Risk by category/severity |
| Top Risks | List | Highest risk areas |
| Control Effectiveness | Score | Control quality assessment |

## UC-RISK-004: Liquidity Risk

### Stakeholder Stories

#### As a Treasury Manager
**I want to** monitor liquidity positions
**So that** I can ensure adequate funding

#### As a Risk Manager
**I want to** stress test liquidity scenarios
**So that** we're prepared for adverse conditions

---

### Input/Process/Output

**Input:**
| Field | Type | Description |
|-------|------|-------------|
| Cash Positions | File | Current cash balances |
| Inflows | File | Expected cash inflows |
| Outflows | File | Expected cash outflows |
| Credit Lines | Data | Available credit facilities |
| Stress Scenario | Dropdown | Normal/Moderate/Severe |

**Process:**
1. Aggregate cash positions
2. Project cash flows
3. Calculate LCR and NSFR
4. Run stress scenarios
5. Generate liquidity gap analysis

**Output:**
| Field | Type | Description |
|-------|------|-------------|
| LCR | Percentage | Liquidity Coverage Ratio |
| NSFR | Percentage | Net Stable Funding Ratio |
| Liquidity Gap | Currency | Funding gap by time bucket |
| Survival Days | Integer | Days of liquidity coverage |
| Stress Results | Matrix | Impact under stress |

## UC-RISK-005: Counterparty Risk

### Stakeholder Stories

#### As a Credit Risk Manager
**I want to** assess counterparty exposure
**So that** I can manage concentration risk

#### As a Trader
**I want to** see real-time counterparty limits
**So that** I can trade within approved bounds

---

### Input/Process/Output

**Input:**
| Field | Type | Description |
|-------|------|-------------|
| Counterparty ID | String | Counterparty identifier |
| Trade Data | File | Current positions with counterparty |
| Collateral Data | File | Collateral posted/received |
| Netting Agreements | Data | Netting set information |
| Market Data | Feed | Current market prices |

**Process:**
1. Calculate current exposure
2. Simulate future exposure (Monte Carlo)
3. Calculate PFE and EPE
4. Apply netting and collateral
5. Calculate CVA

**Output:**
| Field | Type | Description |
|-------|------|-------------|
| Current Exposure | Currency | Mark-to-market exposure |
| PFE | Currency | Potential Future Exposure |
| EPE | Currency | Expected Positive Exposure |
| CVA | Currency | Credit Valuation Adjustment |
| Limit Usage | Percentage | % of approved limit used |
